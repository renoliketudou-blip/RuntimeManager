# RuntimeManager v2.0 Frozen Contract

## 1. Scope

RuntimeManager (RM) only handles runtime container lifecycle and host-path preparation for OpenClaw.

RM is internal-only and synchronous:

- `ensure-running`
- `stop`
- `delete`
- `get-status`

RM does not manage business task state, does not generate IDs, and does not verify DB truth.

## 2. Upstream Request Contract

`POST /internal/runtime-manager/containers/ensure-running` request body:

- required: `userId`, `runtimeId`, `volumeId`, `routeHost`, `retentionPolicy`, `compat`
- optional: `configMount`, `env`, `envOverrides`

`compat` only keeps:

- `openclawConfigDir`
- `openclawWorkspaceDir`

Removed fields in v2.0:

- `imageRef`
- `compat.networkName`
- `compat.gatewayPort`
- `compat.bridgePort`

## 3. Frozen Runtime Baseline

- image: `ghcr.io/openclaw/openclaw@sha256:a5a4c83b773aca85a8ba99cf155f09afa33946c0aa5cc6a9ccb6162738b5da02`
- command: `node dist/index.js gateway --bind lan --port 18789`
- shared network: `clawloops_shared`
- stable alias + endpoint: `rt-<runtimeId>` / `http://rt-<runtimeId>:18789`
- readiness port: only `18789` (18790 kept for compatibility only)

Required env:

- `HOME=/home/node`
- `TERM=xterm-256color`
- `TZ=UTC`
- `OPENAI_BASE_URL=http://litellm:4000` (cannot be overridden)

## 4. Mounting Rules

Base mounts:

- `compat.openclawConfigDir -> /home/node/.openclaw`
- `compat.openclawWorkspaceDir -> /home/node/.openclaw/workspace`

Optional fixed mounts from `configMount`:

- `configFilePath -> /home/node/.openclaw/openclaw.json` (ro)
- `secretFilePath -> /run/clawloops/secrets/gateway.token` (ro)

RM does not parse secret content or auto-convert secret files into env vars.

## 5. State and Errors

Observed states:

- `creating`, `running`, `stopped`, `error`, `deleted`

Not found container fact:

- `GET /containers/{runtimeId}` returns `200` with `observedState=deleted`

Error codes:

- `RUNTIME_CONTRACT_DRIFT` (409)
- `RUNTIME_ACTION_CONFLICT` (409)
- `RUNTIME_START_FAILED` (500/502)
- `RUNTIME_STOP_FAILED` (500)
- `RUNTIME_DELETE_FAILED` (500)

## 6. Drift Policy

When container exists and any key contract item mismatches (image/command/network/labels/mounts/env baseline), RM must return `409 RUNTIME_CONTRACT_DRIFT` and must not recreate silently.

## 7. Migration Notes from v0.1

- All `crewclaw.*` labels are replaced with `clawloops.*`.
- Shared network renamed from `crewclaw_shared` to `clawloops_shared`.
- API payloads using `imageRef` and compat ports/network must be updated.
- `get-status` on missing container is now `200 + deleted`, not 404.
